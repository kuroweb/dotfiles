# 検証コマンド

現在のコードベース状態に対して包括的な検証を実行します。

## 手順

以下の順序で検証を実行:

1. **ビルドチェック**
   - このプロジェクトのビルドコマンドを実行
   - 失敗した場合、エラーを報告して停止

2. **型チェック**
   - TypeScript/型チェッカーを実行
   - ファイル:行の形式ですべてのエラーを報告

3. **Lintチェック**
   - リンターを実行
   - 警告とエラーを報告

4. **テストスイート**
   - すべてのテストを実行
   - 合格/失敗の数を報告
   - カバレッジパーセンテージを報告

5. **Console.log監査**
   - ソースファイル内のconsole.logを検索
   - 場所を報告

6. **Gitステータス**
   - コミットされていない変更を表示
   - 最後のコミット以降に変更されたファイルを表示

## 出力

簡潔な検証レポートを生成:

```
検証結果: [合格/失敗]

ビルド:    [OK/失敗]
型:        [OK/Xエラー]
Lint:      [OK/X件の問題]
テスト:    [X/Y 合格, Z% カバレッジ]
シークレット:  [OK/X件検出]
ログ:      [OK/X件のconsole.log]

PR準備完了: [はい/いいえ]
```

クリティカルな問題がある場合は、修正案とともにリスト化。

## 引数

$ARGUMENTSは以下のいずれか:

- `quick` - ビルド + 型のみ
- `full` - すべてのチェック（デフォルト）
- `pre-commit` - コミットに関連するチェック
- `pre-pr` - 完全なチェック + セキュリティスキャン
